  • 版本控制: Git 安裝與常用指令操作

    # 版本控制: Git 安裝與常用指令操作

    # Mac 系統

    Mac 系統使用內建終端機，可下載色彩配置 Tomorrow Night 並匯入使用。 終端機開啟預設的位置是登入帳號的家目錄，通常以 ~ 符號表示。

    網路上許多教學推薦 iTerm2 這套軟體，有興趣也可以試試。

    附上更多色彩配置可自行選擇或於終端機中設定。

    # Win 系統

    Win 系統請先安裝 git，安裝過程建議全部使用預設值。 安裝完成後於任一資料夾按右鍵，並選擇「Git Bash Here」

    # 檔案與目錄操作常用的指令

    以下的指令在 Mac/Win 上都相同

    # cd = change directory 移動目錄
cd foldername
cd parentFolder/chlidFolder

# 回上一層
cd ..

# 回家目錄
cd ~

# 顯示目前路徑內項目
ls

# 新增資料夾，如有空格請用引號包起來
mkdir foldername
mkdir 'hello world'

# 新增多個資料夾
mkdir css images js

# 新增檔案
touch example.html

# 新增多個檔案
touch reset.css main.css theme.css

# 移動或更換檔名
mv nameA nameB

# 刪除檔案
rm filename

# 刪除資料夾
rm -r folderName
    # 實作練習

    請以指令產生以下結構

    project/
├── css/
│    ├── main.css
│    ├── reset.css
│    └── theme.css
├── images/
│    ├── bg.png
│    └── logo.png
├── js/
│    └── main.js
└── index.html

    # Git 的安裝

    於 Git 官網下載系統對應的安裝程式，依步驟安裝即可。

    以 Mac 系統為例，安裝完成後開啟「終端機」並輸入

    git --version

    若有回饋版本則安裝成功。

    # 設定使用者資訊

    開始使用 Git 前，先設定好使用者資訊，才能記錄提交者資訊喲。 (全域的設定資訊存在使用者家目錄的 .gitconfig 檔案中)

    git config --global user.name <name>
git config --global user.email <email>

    # 新專案，建立 Git Repository

    git init

    執行上述指令，在資料夾中會產生 .git 資料夾，所有與版本控制相關的檔案都存放於此。若在 Mac 系統上看不到可使用快速鍵 cmd + shift + . 切換隱藏檔的顯示或隱藏

    在 Mac 系統中檔案或資料夾只要以 . 開頭即為系統隱藏

    # 版本控制的簡易流程

    簡而言之，流程就是「git init > 新增或修改檔案 > git add > git commit」

    每個步驟都可以使用 git status 來確認目前的狀態，並使用 git log 來查詢提交的記錄。

    本課程總共學習到以下幾個 git 指令

    • git config: 設定 git 參數用。
    • git init: 建立版本控管庫。
    • git add: 將檔案增加到暫存區 (將東西拿到超商櫃台等待寄送)
    • git commit: 填寫訊息並將檔案提交到版本庫中。 (將寄送資訊、備註後將東西寄出)
    • git status: 查詢狀態
    • git log: 查詢提交記錄
    # 新增/修改檔案

    當新增檔案時，新檔案並未列入追蹤，狀態為 untracked

    要將檔案列入追蹤使用

    • git add filename: 加入指定檔案
    • git add .: 一次將目前路徑內所有檔案加入 (含子資料夾)
    • git add --all: 將整個專案資料夾的檔案加入

    而修改檔案一樣使用相同的指令，將檔案加入暫存區。

    # 提交

    提交的意思即為將暫存區裡的檔案，移動到版本記錄庫中。如提交的訊息較少，建議用以下指令進行快速提交

    git commit -m 'this is a message'

    若使用 git commit 指令來進行提交，可輸入多行的訊息，此時預設會以 Vi (編輯器) 來進行操作。

    • 按下 a 鍵，並確認左下角出現 -- INSERT -- 即進入編輯模式，輸入提交訊息。
    • 輸入完成後按下 ESC 鍵，跳出編輯模式
    • 輸入 :wq 存檔並離開即完成提交。
    # 查詢記錄

    提交後可用 git log 來查詢提交的記錄，記錄上會有 SHA1 值、提交者的相關資訊、時間與提交訊息。

    # Sublime

    安裝 Sublime Text 3.2 以上版本與 Git 擴充套件。 可用 cmd+shift+p 開啟命令配盤，並輸入相關指令。如：

    Git Quick Commit (current file) = git add filename + git commit

    # 補充

    Git 不會對資料夾進行控管，如果想要保留特定的資料夾需要自行加入檔案。例：

    # 建立 important 資料夾
mkdir important

# 於資料夾中新增一個 .cd-keep 的系統隱藏檔
touch important/.cd-keep

# 確認是否列入控管
git status

    # 參考資料

    推薦高見龍編寫的「為你自己學 Git」，個人認為目前網路上最易理解的課程。
