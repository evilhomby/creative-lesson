    # 關於 HTML Forms

    Form (表單) 主要作用為提供使用者的輸入資料，通常會配合後端程式 (PHP, NodeJS…等) 進行處理。

    例：一個簡易的登入表單 (範例：post-form.php)

    <form action="login_check.php" method="post" target="_self">
	<label>Username: <input type="text" name="username"></label><br>
	<label>Password: <input type="password" name="password"></label><br>
	<input type="submit" value="Login">
</form>

    # 表單屬性

    • action 屬性來定義資料送到那裡去處理，省略則送到目前的頁面。
    • method 屬性定義資料傳送的方式。
    • target 屬性定義資料傳送的對象，同一視窗、開新分頁或搭配頁框使用。

    # 使用 GET 或是 POST 傳遞資料的差異與使用時機

    當表單省略 method 屬性設定時，預設是以 GET 進行傳遞資料，以下兩個寫法作用相同。

    <form></form>

<form method="get"></form>
    # 使用 GET

    GET 將資料以 name=value 的形式附加在網址上傳遞，因此不適合使用在傳送隱密性資料，例如：密碼。因資料附加上網址上的特性，因此可以做為書籤或是傳送包含資料的網址給其他人。

    例：https://www.google.com/search?q=試管嬰兒

    # 使用 POST

    POST 將資料打包後傳送，不會附加在網址上。安全性比 GET 好，適用在輸入個人或隱密性資料。(傳送的資料仍然可以被查看)

    # 表單內常見要素

    • 表單內常見的要素，例如：<label>, <input>, <select>, <option>, <textarea>, <button>
    • <input> 要素透過 type 屬性來呈現多種不同的形式，例如：文字輸入、密碼、提交按鈕…等

    因表單要素種類與屬性多，建議使用 Emmet 來寫。

    # 重要的屬性

    基本上一個表單欄位會有 name 與 value 兩個屬性用於資料傳送。

    <input type="text" name="keyword" value="試管嬰兒">

    不需傳送值的欄位，如：按鈕。則不需給予 name 屬性。

    <input type="submit" value="按鈕上的文字">

    與資料傳送相關的屬性如下：

    屬性 說明
    name 傳送資料的名稱，若沒有此屬性資料不傳送。
    value 預設傳送資料的值 (字串)
    readonly 唯讀，表單欄位有 name 的情況下仍然會傳送
    disabled 停用，此表單欄位不傳送。

    其他常用的屬性：

    屬性 說明
    placeholder 用於提示的文字
    autofocus 將輸入游標指向表單欄位
    pattern 用於驗證，需搭配規則表達式。例：\d{0,3} 限制輸入 0-3 位的數字
